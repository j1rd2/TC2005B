¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de computadoras conectadas entre si, donde se pueden transferir datos. Al mismo tiempo es una estructura que usa el protocolo IP.

¿Cuáles son las partes de un URL?

Uniform Recourse Locator cuenta con las siguientes partes:
Scheme: Indica el protocolo para acceder al recurso. Los mas comúnes son http (Hypertext Transfer Protocol) y https. Ejemplo: https...


Dominio/Domain: Encuentra el servidor en el que se encuentra el recurso. Ejemplo: example.com


Subdominio/subdomain: Un subdominio que complementa al dominio. Ejemplo: example.mx.com


Ruta/path: Es la ruta que sigue el recurso. Es como se encuentra estrucuturada el elemento. Ejemplo: www.example.com.mx/archive1/archive2.html


Consultra/Query: Parametros que se agregan para agregar informacion extra al servidor. Los espacios usan "?" y son palabras clave. Ejemplo: www.examle.com/buscar?query=ejemplo


Fragmento/fragment: Conocido como hash, hace referencia a una parte especifica de una pagina web.Se indica mediante el símbolo "#", seguido del identificador del fragmento.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Solicita y obtiene datos de un recurso. Cuando se hace una solicitud GET, el servidor devuelve los datos solicitados. Esta solicitud NO debe alerar el contenido.


HEAD: Similat a GET pero este solo solicita encabezados. Util sin tener que descargar todo el contenido.


POST: Envia datos al servidor para que sean procesados. Este método si puede tener efectos secundarios en el servidor.


PUT: Se utiliza para actualizar completamente el recurso de una servidor. Se utiliza para enviar una nueva versión del recurso.


PATCH: Similar a PUT, pero se modifica para aplicar parcialmente modificaciones al recurso.


DELETE: Se utiliza para eliminar un recurso del servidor.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 es unas versión mas moderna, pulida y funciones añadidas de HTML4. Las diferencias principales son:

- Elementos añadidos

- Formularios mejorados

- Soporte multimedia mejorado

- Canvas y SVG

- Almacenamiento local

- Compatibilidad dispositivos mobiles.

- Geolocalizació

- Comptabilidad nuevas tecnologias


¿Qué componentes de estructura y estilo tiene una tabla?

Tiene varios componentes como:

- table: Elemento principal que define una tabla, el resto de elementos se colocan dentro de este.

- tr: Representa una fila de la tabla. Los datos de la fila se colocan dentro de esta.

- td: Representa una celda de datos en una fila.

- th: Una fila de encabezado/titulo de columna o fila.

- thead: Tabla de encabezados. Contiene celdas de encabezado.

- tbody: Cuerpo de la tabla. Contiene celdas de datos de la tabla.

- tfoot: Pie de la tabka. Contiene celdas al final de la tabla.


¿Cuáles son los principales controles de una forma HTML5?

Los controles de un formulario permiten a los usuario interactuar y enviar elementos al servidor. Algunos de los mas importanes son:

- input type="text": Permite al usuario ingresar texto.

- input type="password": Similar al texto pero "cubre" los caracteres.

- input type="email": Permite ingresar direcciones de correo electronico.

- input type="number": Permite ingresar texto.

- input type="tel": Permite ingresar numeros telefonicos.

- input type="url": Valida que el ingreso sea una dirección URL.

- input type="checkbox": Permite seleccionar opciones.

- input type="radio": Similar a checkbox pero permite seleccionar una sola opción.

- input type="file": Permite subir archivos.

- input type="submit": Crea un boton para enviar el formulario al servidor.

- input type="button": Permite crear un boton que puede ejecutar una función por medio de JS.

- input type="reset": Permite restablecer los valores de un formulario.

- textarea: Permite ingresar al usuario texto multilinea.

- select: Permite crear un menú desplegable que permite seleccionar opciónes.

- option: Permite crear una opción dentro de select.

¿Cuál es el ciclo de vida de los sistemas de información?
¿Cuál es el ciclo de desarrollo de sistemas de información?

Ambos pueden ser similarer pero tiene ciertas diferencias.
El ciclo de vida de los sistemas de informacion es la metodologia de el proceso de llevar a cabo el proyecto, pre, durante y post el proyecto.
El ciclo de desarrollo es el proceso para desarrollar el proyecto a partir de un analisis de los requerimientos del proyecto.
